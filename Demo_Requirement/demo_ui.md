# 光储充预测 Demo — 需求文档 v2.0

> **修订说明**：本版剔除了无法获取实时充放电/储能/变压器数据的硬性要求，聚焦于项目已有资源可实现的功能：气象 API 实时数据接入、光伏出力预测、充电负荷预测、历史数据探索分析、电价策略建议。

---

## 一、项目定位

构建一个 **「光储充智能预测 Demo」** Web 应用，面向能源管理场景，展示以下核心能力：

- ☀️ 基于实时气象数据的光伏出力预测
- ⚡ 基于历史模式的充电负荷预测
- 📊 训练数据探索与可视化分析
- 💰 分时电价策略建议
- 🌤️ 实时气象仪表板

**目标用户**：能源管理员、项目演示评审、学术交流。

## 三、页面功能详述

### Tab 1 — 预测总览仪表板（首页）

#### 3.1.1 顶部指标卡片

| 指标 | 说明 |
|------|------|
| ☀️ 光伏峰值预测 (kW) | 未来 N 小时内光伏出力最大值 |
| ⚡ 充电负荷峰值 (kW) | 未来 N 小时内充电负荷最大值 |
| 🌿 绿电替代率 (%) | min(光伏总出力 / 充电总需求 × 100, 100) |

> **数据来源**：光伏预测模型 + 充电负荷预测模型。

#### 3.1.2 电价状态栏

- 显示 **当前电价**（元/kWh）和 **当前时段**（峰/平/谷）
- 内置分时电价表（默认：峰 1.0、平 0.6、谷 0.3 元/kWh，可在配置文件中修改）
- 时段划分可配置

#### 3.1.3 预测控制

- 下拉选择预测时长（6h / 12h / 24h / 48h）
- 「🚀 运行预测」按钮触发预测流水线：
  1. 调用气象 API 获取当前天气
  2. 构造模型输入特征
  3. 光伏模型推理 → 光伏出力序列
  4. 充电模型推理（MC Dropout）→ 负荷均值 + 不确定性区间
  5. 汇总指标 + 生成策略建议

#### 3.1.4 策略建议报告

基于预测结果自动生成 Markdown 格式报告，包含：
- **预测总览**：光伏总出力、充电总需求、绿电替代率
- **供需平衡分析**：光伏盈余/不足时段统计
- **变压器容量检查**：以 `TRANSFORMER_CAPACITY_KW` 为基准，超阈值预警
- **电价引导策略**：建议在谷时 + 光伏盈余时段集中充电
- **风险提示**：天气不确定性提示、建议刷新周期

> ⚠️ 策略建议基于规则引擎，非实时优化调度，仅供参考。

### Tab 2 — 预测曲线（可视化）

#### 3.2.1 光伏出力预测曲线

- 负值（绿色）= 光伏盈余可储能或上网

#### 3.2.4 分时电价背景

- 图表背景按峰/平/谷时段着色，直观展示电价变化

## 四、数据流设计

```
用户点击「运行预测」
       │
       ▼
① 调用气象 API → 获取当前温度/湿度/辐照度等
       │
       ▼
② 构造光伏模型输入特征 (5维: 温度, 湿度, 风速, 辐照度, 时间特征)
       │
       ▼
③ 光伏 CNN-LSTM 模型推理 → solar_forecast[n_steps]
       │
       ▼
④ 构造充电模型输入特征 (15维: 时序滞后特征 + 时间特征 + 电价)
       │
       ▼
⑤ 充电 Hybrid 模型 MC Dropout 推理 → load_mean[n_steps] + load_std[n_steps]
       │
       ▼
⑥ 汇总仪表板指标 + 生成策略建议 → 渲染到 UI
```

## 七、技术约束

1. **全部本地运行**：不依赖外部数据库或云端服务
2. **CPU 兼容**：模型推理优先使用 CUDA，支持 CPU 回退
3. **低延迟**：预测流水线 < 10 秒（MC Dropout 采样 30 次以内）
4. **可配置**：电价、变压器容量、预测时长等参数集中在 `config.py`

## 八、排除范围（本 Demo 不包含）

| 功能 | 排除原因 |
|------|----------|
| 实时充放电功率监控 | 无充电桩数据采集接口 |

| 变压器实时负载率 | 无变压器监测数据 |

| 用户登录/权限管理 | Demo 用途，无需认证 |
